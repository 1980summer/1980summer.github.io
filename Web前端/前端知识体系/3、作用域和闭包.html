    # 1、什么是作用域？ 什么是自由变量？

    A： 作用域包含：全局作用域、函数作用域、块级作用域（es6新增）

    ​ 自由变量：

    • 一个变量在当前作用域没有定义，但被使用了
    • 向上级作用域一层一层一次寻找，知道找到为止
    • 如果找到全局作用域都没有找到，则报错 xx is not defined

    # （四）异步

    # 1、同步和异步的区别是什么？

    A： JS本质上是门单线程语言，只能在同一个时刻做一件事。但是，遇到等待（如网络请求、定时任务等），不能原地卡住，因此就催生出异步这个概念。异步就是为了解决单线程等待的这个问题。异步是基于 callback 函数形式来调用的。

    例如：异步：

    // 异步 （callback 回调函数 ，setTimeout 就是一个callback回调函数）
console.log(100)
setTimeout(() => {
    console.log(200)
}, 1000)
console.log(300)
console.log(400)
// 控制台输出顺序： 100、300、400、200

    # 3、手写promise加载一张图片

     const url ='https://img3.sycdn.imooc.com/5a9fc8070001a82402060220-160-160.jpg'
 function imgLoad (src) {
     // new 一个 Promise实例
     // 实例的括号里面的参数是一个函数，此处是一个箭头函数
     return new Promise (
         // resolve, reject 这两个参数也是函数
         (resolve, reject) => {
             const img = document.createElement('img')
             img.onload = () => {
                 resolve(img)
             }
             img.onerror = () => {
                 const err = new Error(`图片加载失败 ${src}`)
                 reject( err )
             }
             img.src = src
         }
     )
 }
// 调用函数
imgLoad(url).then(img => {
    console.log(img.width);
    return img
}).then(img => {
    console.log(img.height);
}).catch(ex => console.error(ex))

    # 8、Promise有哪三种状态？如何变化？

    ​ A：

    • pending 状态， 不会触发 catch和 then
    • resolved 状态（fulfilled），会触发后续的 then 回调函数
    • rejected 状态，会触发后续的 catch 函数

    ​ ==pending =======> resolved

    ​ ==pending =======> rejected

    注意1：一旦确定了resolved状态或者rejected状态，那么就是不可逆的！！

    注意2：pending不会触发回调，resolved 会触发 then 回调，rejected 会触发 catch 回调。

    注意3：==无论是 then 还是 catch，只要里面没有报错，它返回的永远是一个 resolved 的 Promise，如果报错了那么它返回的永远是一个 rejected 的 Promise==，并且会决定后续调用的是then还是catch

    // 第一题
Promise.resolve().then(() => {
    console.log(1)  // resolved
}).catch(() => {
    console.log(2)
}).then(() => {
    console.log(3) 
})
// 1 3

// 第二题
Promise.resolve().then(() => {
    console.log(1)
    throw new Error('erro1')  // rejected
}).catch(() => {
    console.log(2)            // resolved
}).then(() => {
    console.log(3)
})
// 1 2 3

// 第三题
Promise.resolve().then(() => {
    console.log(1)
    throw new Error('erro1')  // rejected
}).catch(() => {
    console.log(2)   // resolved
}).catch(() => { // 注意这里是 catch
    console.log(3)
})

    # 9、event-loop

    ​ 用图片表示event loop：

    • 每次 call stack 清空（ 即每次轮询结束），即同步任务执行完

    • 都是 DOM 重新渲染的机会，DOM结构如果有改变则重新渲染

    • 然后再次去触发下一次 Event Loop

    # 11、异步的本质

    ​ 虽然async/await是灭异步回调的终极武器，但是JS还是单线程，还是得有异步，还是得基于event-loop。因此，async/await 只是一个语法糖（但这颗糖真香）

    ​ async/await 改变不了异步的本质，await后面的代码，仍旧可以看作是callback的内容（例子在下面的代码里）

    # 13、宏任务/微任务

    • 宏任务：setTimeout、setInterval、Ajax、DOM事件
    • 微任务：Promise async/await
    • 两者区别：
      • 微任务执行时机比宏任务要早
        • 微任务在DOM渲染前触发
        • 宏任务在DOM渲染后触发
    • 微任务是 ES6 语法规定的
    • 宏任务是由浏览器规定的，两者存放的地方不一样

    ####14、描述 even loop 的机制（可画图）

    ​ A:

    • 1 event loop 过程:
    • 2 同步代码，一行一行放在 Call Stack 执行.
    • 3 遇到异步，先“记录”下，等待时机（定时、网络请求等）.
    • 4 时机到了，就移动到 Callback Queue
    • 5 如果 Call Stack 为空（即同步代码执行完），Event Loop 开始工作.
    • 6 轮询查找 Callback Queue，如有则移动到 Call Stack 执行.
    • 7 然后继续轮询查找（永动机一样）
