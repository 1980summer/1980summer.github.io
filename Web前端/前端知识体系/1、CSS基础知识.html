# 2、块状元素和内联元素

答案：

  • 块状元素
    • display：block/table
    • div、h1-h6、 table、ul、ol、p 等
    • 特点是独占一行
  • 内联元素
    • display：inline/inline-block
    • span、img、input、button 等
    • 特点是不会独占一行，会挨着往后排直到浏览器的边缘换行为止

A：

    • 空内容的p标签也会重叠

# 5、margin负值问题

*Q：*对margin 的 top left right bottom设置负值，有何效果?

A：

  • margin-top 和 margin-left 负值，元素分别向上、向左移动
  • margin-right 负值，若有右侧元素，则右侧元素左移，而自身元素不受影响
  • margin-bottom 负值 ，若有下方元素，则下方元素上移，自身元素不受影响。

# 6、什么是BFC？如何应用？

A：

BFC全称 Block Format Context，是一块独立渲染区域，内部元素的渲染不会影响边界以外的元素。

Q: 常见的形成BFC的条件有哪些？

A:

  • float 不是 none
  • position 不是 absolute 或 fixed
  • overflow 不是 visible
  • display 是 flex 或 inline-block

Q： 如何应用？ 举例子

A: 如下，给元素添加一个bfc类，给类设置 overflow：hidden 的样式

<style type="text/css">
        .container {
            background-color: #f1f1f1;
        }
        .left {
            float: left;
        }
        .bfc {
            overflow: hidden; /* 触发元素 BFC */
        }
    </style>
<div class="container bfc">
        <img src="xxx.png" class="left" style="magin-right: 10px;"/>
        <p class="bfc">某一段文字……</p>
</div>

# 7、[默写] 如何实现圣杯布局和双飞翼布局？（必考）

A：

  • 圣杯布局和双飞翼布局的目的（特点）：

    • 三栏布局，中间一栏最先加载和渲染（中间一栏内容最重要）
    • 两侧内容固定，中间内容随着宽度自适应
    • 一般用于PC网页

  • 圣杯布局和双飞翼布局的技术总结：

    • 使用 float 布局
    • 两侧使用 margin 负值，以便中间内容横向重叠

# 9、[默写] flex 实现一个三点的色子

  • 常用语法回顾（必须熟练掌握）
    • flex-direction 主轴方向，有横向、纵向 (不屑这个属性就默认是横向)
    • justify-content 主轴对齐方式，从开始对齐、从结束对齐、从两边对齐、居中对齐
    • align-items 侧轴（即与主轴垂直的那个轴）对齐方式，从开始对齐、从结束对齐、从两边对齐、居中对齐
    • flex-wrap 换行
    • align-self 子元素在其他轴的对齐方式 ，从开始对齐、从结束对齐、从两边对齐、居中对齐

# （二）定位

# 1、absolute 和 relative 分别依据什么定位？

A：

  • relative 依据自身定位
  • absolute 依据父元素中最近一层的定位元素定位
  • 定位元素：absolute、relative、fixed，或者直接是 body

# 2、居中对齐有哪些方式？（必问）

A：

  • 水平居中

    • 行内元素（inline）====> text-align: center
    • 块元素（block）====> margin: auto
    • absolute 元素：====> left:50% + margin-left 负值

  • 垂直居中

    • inline 元素 ====> line-height: height

    • absolute 元素 ====> top: 50% + margin-top 负值 (必须知道子元素的宽高才能使用)

    • absolute 元素 ====> left:50% + top:50%+ transform: translate(-50%, -50%) (不知道子元素的宽高也能使用，但是css3

      的新属性，老版本浏览器可能不支持 )

    • absolute 元素 ====> top、left、bottom、right = 0 + margin: auto (不知道子元素的宽高也能使用，没有兼容性问题)

# 1、line-height 如何继承（有大坑）

*Q：*如下代码， p 标签的行高将会是多少？

 <style type="text/css">
        body {
            font-size: 20px;
            line-height: 200%;
        }
        p {

            font-size: 16px;

  <p>这是一行文字</p>
</body>

A:

如果父元素设置了行高，而子元素没设置，那子元素有三种继承父元素行高的情况：

  • 父元素行高写了具体数值，如 30px， 则子元素继承该值
  • 父元素行高写了比例，如 2 / 1.5， 则子元素继承该比例，用 2 或 1.5 乘以 子元素字体大小
  • 父元素行高设写了百分比***，如 200 %， 则继承计算出来的值，用 200% 乘以父元素*字体大小（考点，坑！！！），所以答案是200% X 20px = 40px

# （四）移动端响应式

# 1、rem 是什么

​ A：

  • rem 是一个长度单位

    • px， 是绝对长度单位，最常用

    • em， 相对长度单位，相对于父元素，不常用

    • rem，相对长度单位，相对于根元素（root，所以叫rem），常用于响应式布局 （假如根元素字体大小是100px，这100px就是 1rem）

div {

    margin-top: 10px;
    font-size: 0.16rem;  <!-- 0.16 X 100 = 16px -->
    }
</style>

# 2 、响应式布局的常见方案

​ media query（css3中的新属性），根据不同的屏幕宽度设置根元素 font-size

​ rem，基于根元素的相对定位

  • <style type="text/css">
    /* 如果面试时写不出来这个 @media的 代码，也没有关系， 实际开发中一年写个一两次都算多的了 */
        @media only screen and (max-width: 374px) {
            /* iphone5 或者更小的尺寸，以 iphone5 的宽度（320px）比例设置 font-size */
            html {
                font-size: 86px;
            }
        }
        @media only screen and (min-width: 375px) and (max-width: 413px) {
            /* iphone6/7/8 和 iphone x */
            html {
                font-size: 100px;
            }
        }
        @media only screen and (min-width: 414px) {
            /* iphone6p 或者更大的尺寸，以 iphone6p 的宽度（414px）比例设置 font-size */
            html {
                font-size: 110px;
            }
        }
        body {
            font-size: 0.16rem;
        }
        #div1 {
            width: 1rem;
            background-color: #ccc;
        }
    </style>
